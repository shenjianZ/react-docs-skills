# 双语文档编写

默认假设 docs 项目是中英双语站点。

## 默认原则

除非用户明确说明只维护单语内容，否则默认同时考虑：

- `public/docs/zh-cn/`
- `public/docs/en/`

但实际编写顺序优先采用：

1. 先完成中文文档
2. 先审核中文文档内容、结构、标题和路径是否正确
3. 中文确认无误后，再继续编写英文文档
4. 英文文档应基于已确认的中文文档做等价翻译，不要先独立编写英文内容

## 编写约定

- 两种语言的目录结构尽量保持一致
- 文档 slug 尽量保持一致，便于语言切换和路由维护
- 中文文档是英文文档的前置版本，先确认中文，再扩展到英文
- 英文文档应以已确认中文文档为基准做等价翻译，避免中英文内容范围、结构、标题层级不一致
- 如果只更新了其中一种语言，要提醒用户另一种语言可能已经不同步
- 如果配置文件里引用了某个文档路径，要确认对应语言目录里确实有这个文件

## 什么时候提醒配置同步

如果用户说"页面写好了但导航里没有"，就提醒他检查：

- `site.yaml`
- `site.en.yaml`

文档文件和配置文件通常需要一起维护。
