# React Docs UI 脚手架使用

当用户询问如何使用 `create-react-docs-ui` 创建、启动、理解一个文档项目时，使用这个 skill。

默认假设用户正在操作一个已经生成好的 docs 项目，例如 `D:\tmp\test-docs\docs`。

## 适用场景

- 用户想创建一个新的 docs 站点
- 用户想知道生成后的项目目录该怎么看
- 用户想知道安装依赖、启动开发环境该执行什么命令
- 用户想知道项目创建后应该先改哪些文件

不要用这个 skill 回答 `site.yaml` 字段设计细节，或 Markdown / MDX 编写规范。那类问题交给：

- `react-docs-ui-config`
- `react-docs-ui-doc-authoring`

## 工作流程

1. 先确认用户讨论的是 `create-react-docs-ui` 生成出来的项目。
2. 从用户的 docs 项目根目录视角解释项目结构。
3. 只讲用户在项目里实际会执行的命令和会修改的文件。
4. 如果用户想让某个页面出现在导航栏或侧边栏，明确说明只创建文档文件还不够，还要同步修改配置。
5. 如果问题已经进入配置写法或文档写法细节，切换到对应 skill。

## 输出约束

- 回答保持面向最终用户。
- 聚焦"用户在生成后的 docs 项目里该怎么做"。
- 尽量直接引用具体路径，例如 `public/config/site.yaml`、`public/docs/zh-cn/`、`src/components/`。

## 参考文件

- 需要说明创建和启动流程时，读取 `references/scaffold-workflow.md`
- 需要说明项目目录结构时，读取 `references/project-layout.md`
