# React Docs UI 配置编写

当用户询问如何编写或修改 docs 项目中的 `public/config/site.yaml` 或 `public/config/site.en.yaml` 时，使用这个 skill。

默认假设用户正在一个生成后的 docs 项目中工作，例如 `D:\tmp\test-docs\docs`。

## 适用场景

- 用户想知道 `site.yaml` 或 `site.en.yaml` 该怎么写
- 用户想知道某个配置块是做什么的
- 用户想配置导航、侧边栏、主题、搜索、导出、字体、MDX、AI、PWA 等能力
- 用户想把某个页面加到 navbar 或 sidebar

不要用这个 skill 回答脚手架初始化步骤，或 Markdown / MDX 文档内容本身的写法。

## 工作流程

1. 先识别用户要修改的是哪个配置区域。
2. 明确告诉用户应该改哪个文件。
3. 用通俗的话解释这个配置会影响什么。
4. 优先给最小 YAML 片段，除非用户明确要整份配置示例。
5. 说明这个改动是否也应该同步到另一份语言配置文件。
6. 如果问题本质上变成了文档目录、frontmatter 或 MDX 写法问题，转给 `react-docs-ui-doc-authoring`。

## 输出约束

- 回答保持面向最终用户、可直接操作。
- 重点说明"改哪个文件、怎么写、会影响哪里"。
- 回答 `navbar` 和 `sidebar` 问题时，要先区分职责：`navbar` 管顶部导航项，`sidebar` 管每个导航项下文档区域的侧边栏导航。还要明确检查 `navbar.items.link` 是否和 `sidebar.collections` 的 key 及该区域下的文档路径对应起来。
- 项目默认按双语站点考虑，因此通常要同时提醒 `site.yaml` 和 `site.en.yaml`。

## 参考文件

- 需要概览配置块或常见写法时，读取 `references/config-fields.md`
- 需要提醒双语同步、路径一致性时，读取 `references/config-sync-checklist.md`
